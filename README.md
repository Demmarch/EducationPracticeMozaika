# Что это?

Это проект для учебной практики, который выполнен по ТЗ, которое я сам составил.

## Что в репо?

Тут в репозитории представлены различные вариации реализации проекта с учебной практики. В каждой ветке своя реализация.

## Что находится в ветках?

Абсолютно в каждой ветки находятся файлы для бд, можно либо восстановить через db_backup.sql либо через запросы в папке [Файлы для БД](./Файлы%20для%20БД/). В наименовании веток s - значит "сервер", а c - "клиент"

На момент написания есть следующие ветки:
* [sqtcp/csocket](https://github.com/Demmarch/EducationPracticeMozaika/tree/sqtcp/csocket) - Ветка, в которой бэкэнд написан на С++ с использованием QTcpServer, а клиент - на сокетах Dart Flutter. Формат (протокол) общения между клиентом и сервером - Json + '\n' в качестве конца сообщения;

Планируются:
* scrow/chttp - Бэк на C++ с Crow, клиент на htpp протоколе Dart Flutter
* sdrogon/chttp - Бэк на C++ с Drogon, клиент на htpp протоколе Dart Flutter
* suserver/chttp - Бэк на C++ с userver, клиент на htpp протоколе Dart Flutter
* sactix/chttp - Бэк на Rust с Actix Web, клиент на htpp протоколе Dart Flutter

### Дополнительно

Дополнительно планируется написать тестер для всех этих веток. Тестер будет проводить нагрузку на сервер и собирать статистику.
